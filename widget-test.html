2 · Calendar

Booking Calendar

Just the calendar + availability + booking form. No product description or photos — assumes your page already sells the product. Embeds inline (no modal). This is likely the best fit for the contact.html booking hub.

Likely use: contact.html booking hub, one per tab

3 · Product Page

Full Product Page

Complete widget: Bokun photos, description text, itinerary, pricing, and the calendar — all from your Bokun product data. Useful if you want Bokun to manage content. Risk: generic layout may clash with brand.

Possible use: standalone booking landing pages — evaluate carefully

Layout Question

How to show multiple experiences per category?

Golf Cart has 7 experiences. Vespa has 2. Food Tours has 3. Each category page needs a way to present them. Three options are shown below — pick the one that feels right.

Option A Tab Selector

One page for Golf Cart. Tabs across the top — click a tour name to load its Bokun widget below. Only one widget loads at a time.

← Bokun Product Page widget for "Rome Highlights" loads here

Pro: One clean URL per category (golf-cart.html). Easy to link to.   Con: 7 tabs is crowded on mobile. User must read tab names to find what they want.
Option B Stacked on One Page

One page for Golf Cart. All 7 Bokun Product Page widgets stacked vertically — scroll down to see each one.

Bokun Product Page — Rome Highlights (full widget: photos, description, calendar)
Bokun Product Page — Top Ten Landmarks
Bokun Product Page — Catacomb & Appian Way
+ 4 more widgets below…
Pro: Everything visible at once, no clicking.   Con: Very long page. 7 widgets all loading = slow. Hard to navigate.
Option C Separate Page per Experience

Each of the 18 experiences gets its own dedicated page. Category page (golf-cart.html) becomes a card grid linking to each sub-page. One Bokun widget per page = fastest loading.
